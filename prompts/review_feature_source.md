# feature 実装直後のソースレビューを行ってください

このプロンプトは、`prompts/implement_feature.md` で作成した feature 実装ファイルと feature 単体テストファイルを対象に、**実装直後の中間チェック**を行うためのものです。

問題を見つけても、このプロンプトでファイルを修正しません。修正候補としてチャットで報告するだけです。

`25_review_result.md` は作成しません。正式なレビュー結果ファイルは、従来どおり `prompts/review_feature.md` で作成してください。

---

## review_feature.md との違い

| | review_feature_source.md（このプロンプト） | review_feature.md |
|---|---|---|
| 位置づけ | 実装直後の中間チェック | feature 単体の正式レビュー |
| 主な対象 | feature 実装ファイルと feature 単体テストファイル | 仕様、設計、実装、単体テスト、レビュー観点の整合 |
| 修正 | しない | しない（指摘のみ） |
| 出力先 | チャットで報告する | `25_review_result.md` に記録する |

---

## 利用者が指定する項目

以下を対象 feature に合わせて指定してください。

- 対象機能フォルダ: `docs/<command_or_app_name>/features/<feature_name>/`
- コマンド/アプリ名: `<command_or_app_name>`
- 対象機能名: `<feature_name>`
- 実装ファイル: `src/<command_or_app_name>/features/<feature_name>.py`
- テストファイル: `tests/<command_or_app_name>/features/test_<feature_name>.py`
- 補足条件: `必要に応じて書く。なければ「なし」`

## 作業手順

1. `AGENTS.md` を確認してください
2. 対象 feature の `20_spec.md` から `24_review_checklist.md` を確認してください
3. feature 実装ファイルを確認してください
4. feature 単体テストファイルを確認してください
5. 以下の確認観点に沿って、実装とテストを照合してください
6. チャットでソースレビュー結果を報告してください

---

## 確認観点

以下の観点で確認してください。

**仕様・設計との整合**

- 仕様にない便利機能を追加していないか
- `20_spec.md` の仕様と実装が合っているか
- `21_design.md` にない関数や責務を増やしていないか
- `22_flow.md` の呼び出し関係と実装が合っているか
- エラー処理や境界値が仕様・テスト計画とズレていないか

**責務分担**

- CLI引数解析や標準出力など、entrypoint の責務が feature に混ざっていないか
- feature 固有ロジックが entrypoint 側に逃げていないか
- `src/common/` に勝手に切り出していないか
- 標準ライブラリ以外を勝手に使っていないか

**テストの妥当性**

- `23_test_plan.md` の観点を単体テストが満たしているか
- `24_review_checklist.md` の観点に照らして大きな懸念がないか
- テストが仕様ではなく実装詳細に寄りすぎていないか
- テストが実装の誤りを追認する形になっていないか

**範囲外変更**

- AIが指示範囲外のついで修正をしていないか
- 指定されていないファイルが変更されていないか

## 出力形式

以下の形式でチャットに報告してください。

---

### 1. 総合判定

以下のいずれかを記載してください。

- `テスト実行に進めてよい` — 大きな問題なし
- `修正候補あり` — 軽微な懸念または要修正候補がある
- `要人間確認` — 仕様確認や判断が必要な点がある

### 2. 確認したファイル

参照したファイルの一覧を記載してください。

### 3. ソースレビュー結果

各観点の結果を以下の区分で記載してください。

- 問題なし
- 軽微な懸念
- 要修正候補
- 要仕様確認

### 4. 実装ファイルに関する指摘

実装ファイルへの具体的な指摘があれば記載してください。問題なしの場合もその旨を記載してください。

### 5. 単体テストファイルに関する指摘

テストファイルへの具体的な指摘があれば記載してください。問題なしの場合もその旨を記載してください。

### 6. 仕様・設計・テスト計画とのズレ

`20_spec.md`、`21_design.md`、`22_flow.md`、`23_test_plan.md` とのズレがあれば記載してください。

### 7. ついで修正・範囲外変更の有無

指示範囲外の変更があれば記載してください。問題なしの場合もその旨を記載してください。

### 8. 修正する場合の推奨対応

修正候補がある場合、どのように修正すべきかを記載してください。

**このプロンプトでは修正しません。** 修正は、人間が反映対象を判断した後、別作業としてAIに依頼してください。

### 9. 次に進む場合の候補

- 問題なしの場合: `python -m pytest` を実行する
- 修正候補がある場合: 人間が反映対象を判断し、別作業としてAIに修正依頼する
- 仕様確認が必要な場合: `20_spec.md` または `21_design.md` に戻る

---

## 禁止事項

- feature 実装ファイルの修正
- feature 単体テストファイルの修正
- docs の修正
- `tasks.md` の更新
- `25_review_result.md` の作成
- `src/common/` への切り出し
- `python -m pytest` の実行
- 自動修正

このプロンプトは、ソースレビューと修正候補の報告だけを行います。
