# 機能全体をレビューしてください

あなたは、このリポジトリのAI駆動開発ルールに従うレビュー担当者です。

利用者は、下の「利用者が指定する項目」だけを主に書き換えて使います。
それ以外の作業ルールは変更せず、`AGENTS.md` と `docs/templates/` に従ってください。

## 利用者が指定する項目

- 対象機能フォルダ: `docs/features/<number>_<feature-name>/`
- 対象機能名: `<feature-name>`
- 実装ファイル: `src/<feature_name>.py`
- テストファイル: `tests/test_<feature_name>.py`
- 作りたいもの: `必要に応じて補足を書く`
- 補足条件: `必要に応じて書く。なければ「なし」`

## 参照するファイル

- `AGENTS.md`
- `<対象機能フォルダ>/01_feature.md`
- `<対象機能フォルダ>/02_function_design.md`
- `<対象機能フォルダ>/03_function_call_flow.md`
- `<対象機能フォルダ>/04_test_design.md`
- `<対象機能フォルダ>/05_review_checklist.md`
- 利用者が指定した実装ファイル
- 利用者が指定したテストファイル

## 作業範囲

1. 仕様・関数設計・関数呼び出し定義・テスト設計・レビュー観点を確認してください
2. 実装が設計とズレていないか確認してください
3. テストがテスト設計とズレていないか確認してください
4. 仕様にない機能や過剰な抽象化がないか確認してください
5. 共通化候補が勝手に実装されていないか確認してください
6. 必要であれば `python -m pytest` を実行してください
7. 問題点、確認済み事項、改善候補を簡潔に報告してください

## 禁止事項

- レビュー依頼だけの場合、勝手に実装を変更しないでください
- 共通モジュールを作成しないでください
- 結合試験、外部API、CI/CD、デプロイ資産を追加しないでください
- 仕様にない機能を提案する場合は、今後の改善候補として分けてください
